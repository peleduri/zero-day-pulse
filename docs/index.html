✨ Parallel AI Enrichment
Technical DetailsIndirect Prompt Injection (IPI) occurs when an AI system processes external content (a website, email, document, image, or metadata) that contains malicious instructions. When the AI reads this 'poisoned' content, the hidden instructions override or supplement the original system prompt, causing the model to silently follow attacker commands instead of the user's intent. Observed embedding techniques include: visually invisible text (shrinking text to a single pixel, draining color to near-transparency, or using HTML 'hidden' / display:none attributes); burying payloads inside HTML comments and page metadata (meta tags, JSON-LD); meta-tag namespace injection combined with persuasion amplifiers (e.g., the keyword 'ultrathink'); hidden instructions inside image tags and JavaScript. Targets include LLM-powered browsers (Atlas, Comet) and AI agents integrated with email, documents, and third-party tools/plugins, where successful injection can lead to unauthorized actions such as data exfiltration, file deletion, credential theft, or financial fraud.

✨ Parallel AI Enrichment
Technical DetailsIndirect Prompt Injection (IPI) exploits the trust boundary between an LLM's system/user prompt and the external data the LLM retrieves while answering a query. An attacker plants malicious natural-language (or encoded) instructions inside content the LLM will later read — emails, shared documents, calendar invites, web pages, HTML metadata, document footers, etc. When the user issues a benign query, the LLM pulls the poisoned content into its context window (often via Retrieval-Augmented Generation), cannot reliably distinguish user commands from data-borne instructions, and obeys the injected commands. Attacks can be zero-click (no user interaction required beyond a normal search/summarise, as in GeminiJack) and can target agentic capabilities (tool/function calling, code execution, email send, file deletion, payments). Obfuscation techniques observed in the wild include CSS hiding (display:none, visibility:hidden, opacity:0, zero-sized text, off-screen positioning), white-on-white text, HTML comments, XML/SVG CDATA sections, data-* attribute cloaking, base64-encoded payloads decoded at runtime, and meta-tag namespace injection. Data exfiltration is typically achieved by coercing the model to issue HTTP requests (e.g., image tags) to attacker-controlled servers.
Affected ProductsGoogle Workspace with Gemini (Gmail, Docs, Calendar and other Workspace surfaces reachable by Gemini), Google Gemini Enterprise, Google Vertex AI Search (VAIS), Google Gemini CLI. Specific Gemini model versions referenced in research: Gemini 2.0, Gemini 1.5 Pro, Gemini 1.5 Flash. No precise build numbers were published.

(5) End-user hardening — security mitigation notifications surfaced to Workspace admins and users. (6) Third-party hardening advice: treat all retrieved/RAG content as untrusted, strip/sanitise HTML and hidden text, require human-in-the-loop confirmation for any tool/function call that touches email, shell, payments or file systems, restrict outbound egress from agent runtimes, and segment agent tools by trust level.

✨ Parallel AI Enrichment
Technical DetailsIndirect prompt injection is a class of attack in which an adversary embeds hidden malicious instructions inside external content that an AI assistant ingests on the user's behalf (e.g., emails, documents, calendar invites, web pages, images). When the LLM processes that content together with the user's prompt, it cannot reliably distinguish attacker-controlled instructions from legitimate user intent, allowing the attacker to coerce the model into exfiltrating sensitive data (location, memory, contacts, calendar details), invoking tools/connectors without authorization, modifying or deleting user data, or performing other rogue actions. Unlike direct prompt injection (where the attacker types the malicious prompt themselves), the attack is delivered through benign-looking data sources the user has asked the AI to summarize or act on. Real-world examples include: (1) a calendar invite whose description contains a natural-language payload instructing Gemini to summarize meetings and call Calendar.create, leaking private schedule data (Miggo); (2) a malicious web page that uses JavaScript top-level navigation to seed fake queries into a user's Gemini search-history personalization model, then exfiltrates location/name/memory via markdown link rendering (Tenable TRA-2025-23).

Google Workspace administrators are advised to keep Gemini features enabled with the default controls and to follow Google's Workspace admin guidance for indirect prompt injection.

✨ Parallel AI Enrichment
Technical DetailsPRC state-sponsored APT actors gain initial access by exploiting known CVEs in network edge devices (backbone/PE/CE routers, Ivanti Connect Secure, Palo Alto GlobalProtect, Cisco IOS XE) and use VPS hosts and compromised routers as stepping stones. Persistence techniques include modifying access control lists (often naming ACLs 'access-list 20'), installing SSH backdoors/authorized keys on non-standard ports, using SNMP SET requests for persistent access, establishing GRE/IPsec tunnels, and abusing the Cisco IOS XE Guest Shell container to run Python scripts (e.g., 'siet.py', 'dohost'). Credential theft is performed by enabling Cisco Embedded Packet Capture (EPC) to record TACACS+ and RADIUS authentication traffic (e.g., files like 'mycap.pcap', 'tac.pcap'), then decrypting TACACS+ packet bodies using shared secrets to recover administrator credentials; actors also modify TACACS+ server configurations to redirect authentication to attacker-controlled infrastructure. Actors enable SPAN/RSPAN/ERSPAN traffic mirroring, harvest BGP/RSVP/MIB/MPLS configuration via TFTP, brute-force or decrypt Cisco Type 5 and Type 7 passwords, and create new privileged accounts.
