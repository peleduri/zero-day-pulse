✨ Parallel AI Enrichment
Technical DetailsCVE-2024-57727 is a path traversal vulnerability (CWE-22) in SimpleHelp RMM versions 5.5.7 and earlier. Unauthenticated remote attackers can submit crafted HTTP requests containing directory-traversal sequences (e.g., '../') in URL paths or parameters to the SimpleHelp web interface (notably the file/toolbox endpoint) to read arbitrary files from the host. Sensitive files accessible include the server configuration file (configuration/serverconfig.xml) containing hashed admin/user passwords and cryptographic secrets, as well as system files such as /etc/passwd and SSH private keys. The flaw can be chained with CVE-2024-57726 (missing authorization in an admin API) and CVE-2024-57728 (admin-only arbitrary file upload / zip-slip) to escalate from unauthenticated file read to full remote code execution on the SimpleHelp server.

Technical DetailsIndirect Prompt Injection (IPI) occurs when an AI system ingests third-party content containing hidden adversarial instructions that the model treats as legitimate commands. Mitigating prompt injection attacks with a layered defense strategy

No CVE · Google Security Blog · 2025-06-13

Posted by Adam Gavish, Google GenAI Security Team With the rapid adoption of generative AI, a new wave of threats is emerging across the industry with the aim of manipulating the AI systems themselves. One such emerging attack vector is indirect prompt injections. Unlike direct prompt injections, where an attacker directly inputs malicious commands into a prompt, indirect prompt injections involve…

✨ Parallel AI Enrichment
Technical DetailsIndirect prompt injection is an attack in which untrusted external content consumed by an LLM/agent (e.g., the body of an email in Gmail, text in a Google Doc, a calendar invite description, a file in Google Drive, or a message in Google Chat) embeds hidden or out-of-band instructions — such as HTML white-text, zero-font text, encoded payloads, or instructions inside images/documents. When Gemini in Workspace is asked to summarize, reply to, or act on that content, the model can be steered away from the user's task and induced to perform attacker-chosen actions: exfiltrating data from prior emails or Drive files, following links to attacker-controlled URLs, or surfacing crafted phishing/credential-harvesting content to the user. Unlike direct prompt injection (user-to-model), the attacker controls the data channel, not the prompt. The risk is amplified in agentic settings where Gemini can invoke tools to read additional mail/files or trigger external actions, because instructions in one piece of content can cascade across the user's data.
Affected ProductsGemini app (consumer), Gemini in Google Workspace: Gmail, Google Docs editors (Docs, Sheets, Slides), Google Drive, and Google Chat. Google has not published specific build or version numbers; affected deployments are identified at the product/surface level (the Gemini integrations within these Workspace apps).
CVSS ScoreCVSS score unavailable
CVSS VectorCVSS vector unavailable. Indirect prompt injection is a class-of-vulnerability affecting LLM-integrated products; no individual CVE or CVSS v3 base vector has been published for the Gemini/Gmail/Docs/Drive/Chat issues addressed by Google's layered defenses.
Exploit Availabletrue. Multiple public PoCs exist: (1) 0DIN 'Phishing for Gemini' demonstrates indirect prompt injection against Gemini in Gmail/Workspace via hidden white-text email instructions (https://0din.ai/blog/phishing-for-gemini); (2) SafeBreach Labs disclosed a Gemini voice assistant prompt injection exploit (https://www.safebreach.com/blog/gemini-voice-assistant-prompt-injection-exploit/); (3) HiddenLayer reported Gemini for Workspace vulnerability enabling phishing/content manipulation (https://www.hiddenlayer.com/research/new-gemini-for-workspace-vulnerability-enabling-phishing-content-manipulation).
Patch Availabletrue. Google's layered mitigations are being progressively rolled out across Gemini in Workspace (Gmail, Docs editors, Drive, Chat) and the Gemini app. There is no traditional software patch/CVE because the protection is a defense-in-depth architecture rather than a single fix. Advisory URL: https://security.googleblog.com/2025/06/mitigating-prompt-injection-attacks.html
Active Exploitationtrue. Indirect prompt injection attacks against LLM agents, including those targeting Gemini, have been observed in the wild. Google's own blog (Jun 13, 2025) notes that the layered protections have been actively mitigating indirect prompt injection attempts against Gemini, indicating real-world attack activity rather than purely theoretical risk. Unit 42 has documented web-based indirect prompt injection attacks observed against AI agents in real-world settings (https://unit42.paloaltonetworks.com/ai-agent-prompt-injection/).
Threat ActorsNone known. Google's disclosure of this layered defense strategy does not attribute the indirect prompt injection technique to any specific named threat actor group. Separately, Google Threat Intelligence Group has reported that Iranian (APT42), Chinese (APT41), North Korean, and Russian state-aligned actors have abused Gemini for productivity tasks, reconnaissance, and payload development, but those reports do not document those actors exploiting indirect prompt injection against Gemini itself.
MitigationUntil model-level robustness fully solves the issue, Google recommends and has rolled out a layered defense strategy for Gemini in Workspace: (1) prompt-injection content classifiers that screen untrusted content for injection attempts; (2) 'security thought reinforcement' via system prompts/model hardening so the model resists being hijacked; (3) markdown sanitization plus suspicious-URL redaction (URLs flagged by Google Safe Browsing are stripped or rendered as plain text); (4) a Human-In-The-Loop (HITL) confirmation framework requiring user approval before sensitive actions; and (5) end-user security notifications that warn users when suspicious or potentially unsafe AI behavior is detected. Administrators can review these protections in Gemini Workspace settings (e.g., 'Workspace smart feature' controls and Gemini classification settings).
