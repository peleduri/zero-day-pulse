CVE-2024-57727 · CISA US-CERT Alerts · Thu, 12 Ju

Summary The Cybersecurity and Infrastructure Security Agency (CISA) is releasing this advisory in response to ransomware actors leveraging unpatched instances of a vulnerability in SimpleHelp Remote Monitoring and Management (RMM) to compromise customers of a utility billing software provider. ✨ Parallel AI Enrichment
Technical DetailsCVE-2024-57727 is an unauthenticated path traversal vulnerability (CWE-22) in the SimpleHelp RMM web application's toolbox/file-download functionality. The application fails to adequately sanitize user-supplied path input, enabling remote unauthenticated attackers to craft HTTP requests containing directory-traversal sequences (e.g., '../') to download arbitrary files from the SimpleHelp host. Sensitive files exposed include server configuration files (serverconfig.xml) containing plaintext/hashed credentials, hashed user passwords, and—on Linux systems—system files such as /etc/passwd and SSH private keys (e.g., /root/.ssh/id_rsa). The vulnerability is network-exploitable, requires no authentication and no user interaction, and results in high confidentiality impact with no integrity or availability impact. It was part of a chainable set of three SimpleHelp CVEs disclosed in January 2025.
Affected ProductsSimpleHelp Remote Monitoring and Management (RMM) versions 5.5.7 and earlier (v5.4.x line patched via v5.4.10)

Technical DetailsIndirect Prompt Injection (IPI) occurs when an LLM/AI agent ingests untrusted external content (websites, emails, documents, calendar invites, image alt-text, chat messages, payment flows, retrieved documents) poisoned with hidden or overt instructions. The model cannot reliably separate untrusted user/tool data from trusted system instructions, so it follows injected instructions as if they were legitimate commands. Per the primary Google blog and corroborating research (Forcepoint, Unit42, HelpNetSecurity), delivery techniques include: invisible text via CSS ('display:none', 1px font, transparent color, off-screen positioning), white text on white backgrounds, HTML comments, 'aria-hidden' / 'visually-hidden' accessibility classes, custom metadata namespaces (e.g., 'ai:action' meta tags), HTML/SVG CDATA encapsulation, invisible Unicode characters, multi-language / multi-step payloads, terminal-command injection ('sudo rm -rf'), OAuth/tool-call chaining via MCP tool descriptions, and adaptive/streaming payloads. Realized goals observed include data exfiltration to attacker-controlled URLs, financial transaction redirection (PayPal/Stripe donation scams), unauthorized file deletion, AI ad-review evasion (Unit42 case against reviewerpress[.]com), tone override, prompt-token seizure for DoS, brand hijack / SEO manipulation, and CVSS-9-class zero-click exfiltration of tenant data in Copilot-class agents.
Affected ProductsIndirect Prompt Injection as a class affects AI assistants and agents broadly. Specifically affected or named in/around this report: Google Gemini app, Gemini 2.5 models, Workspace with Gemini (Gmail, Docs), Google AI Overviews / Search, Common Crawl corpus; Microsoft 365 Copilot (related CVE-2025-32711 'EchoLeak'); ChatGPT (chatgpt.com); Google Gemini Enterprise (GeminiJack zero-click by Noma Labs); Model Context Protocol (MCP) tools/servers in agentic stacks including Copilot Studio, Azure AI Foundry, and Dataverse MCP. Specific version ranges were not enumerated in the Google blog.

✨ Parallel AI Enrichment
Technical DetailsIndirect prompt injection (IPI) is a class of attack against LLM-backed applications (e.g., Workspace with Gemini) where an adversary embeds malicious natural-language instructions inside a data source or tool that the LLM consumes while completing the user's request (e.g., an email body, Google Doc/Slide, Drive file, web page, or a Cloud log line). The LLM treats those untrusted instructions as legitimate directives, causing it to ignore the user's intent and instead (a) generate attacker-controlled output (phishing messages, malicious URLs, fake security alerts), (b) take unauthorized actions through connected tools (sending mail, modifying Drive/Docs assets, invoking Cloud integrations), or (c) exfiltrate conversation or workspace data. Concretely demonstrated techniques include hidden HTML/CSS (zero-sized text, display:none, opacity:0, color camouflage), SVG/CDATA cloaking, JavaScript-rendered prompts, and URL fragment (#) injection (e.g., 'HashJack'). Per Google's blog, IPI can succeed even without any direct user input, and it is amplified by agentic automation that combines many tools and data sources.
Affected ProductsGoogle Workspace with Gemini (Gemini app, plus Gemini integrations in Gmail, Google Docs, Google Slides, Google Drive, and Google Chat); Gemini 2.5 models; Google Cloud Gemini Cloud Assist. (Indirect prompt injection is a class of attack against AI applications with multiple data sources and agentic tooling; beyond Workspace, it broadly impacts LLM-backed agents that ingest untrusted data from web pages, logs, and third-party tools.)

Specific product-level defects are addressed individually — for example, Tenable TRA-2025-10 in Gemini Cloud Assist was fixed by Google (hyperlink rendering removed from log-summarization responses), CVE-2025-54131 affecting Cursor < 1.3 was fixed in Cursor 1.3, and Gemini-in-Workspace behaviors reported by HiddenLayer were followed by Google's mitigations (the bug was reported and ultimately addressed rather than land as a CVE). Earlier, HiddenLayer publicly demonstrated a working Gemini-in-Workspace IPI PoC in September 2024. Sources: https://unit42.paloaltonetworks.com/ai-agent-prompt-injection/ ; http://security.googleblog.com/2025/06/mitigating-prompt-injection-attacks.html ; https://www.hiddenlayer.com/research/new-gemini-for-workspace-vulnerability-enabling-phishing-content-manipulation
Threat ActorsNone known. MitigationGoogle's stated mitigation strategy is layered rather than a single patch: (1) Deterministic / system-level defenses: markdown sanitization, suspicious-URL redaction via Google Safe Browsing, a User-Confirmation framework that requires explicit user approval for risky operations, tool-chaining policies, and a centralized Policy Engine that performs regex-based takedowns of known-bad patterns. (2) ML-based defenses: proprietary prompt-injection content classifiers trained on synthetic data generated via the Simula harness and on real-world samples. (3) LLM-based defenses: targeted security 'thought reinforcement' — instructions appended around prompt content to steer Gemini to ignore adversarial commands. (4) Model hardening: adversarial robustness testing and adversarial-data retraining of Gemini 2.5. (5) End-user mitigations: security notifications surfaced to users when IPI is detected and contained. Google also augments these defenses via its AI Vulnerability Reward Program (e.g., a $15,000 prompt-injection bug bounty paid in March 2026, indicating third-party reporting also feeds the defense pipeline). For specific related CVEs chained with IPI (e.g., CVE-2025-54131 in Cursor version < 1.3), vendors have shipped dedicated version fixes.

Technical DetailsCVE-2025-48530 is a linear/heap buffer overflow (out-of-bounds access, CWE-125) in the CrabbyAVIF AVIF image parser/decoder (AOSP bug A-419563680). In multiple locations there is a possible condition that results in OOB accesses due to an incorrect bounds check — affecting YUV planes (alpha, Y, and UV planes), chroma width calculation, plane size calculation, and row bytes when decoding crafted AVIF images. A remote attacker could deliver a malicious AVIF (e.g., via a web page, messaging app, or other image-rendering surface) to trigger the OOB access in the System component, potentially leading to remote code execution when chained with other vulnerabilities, with no additional execution privileges required (severity Critical in Google's bulletin). The bug originated from an unsafe Rust code block in the parser, marking the first Rust-based memory-safety vulnerability discovered in Android.

✨ Parallel AI Enrichment
Technical DetailsIndirect prompt injection (IPI) attacks exploit the architecture of generative AI assistants like Gemini, which ingest external/untrusted content (emails, documents, calendar invites, web pages) as part of fulfilling user requests. Attackers embed hidden natural-language instructions within that untrusted content (e.g., in a calendar event description, HTML email with hidden text, or speaker notes in a document). When Gemini processes this content, the adversarial instructions are concatenated with the legitimate system/user prompt, causing the model to follow attacker-dictated actions instead of the user's genuine intent. Documented attack goals include: data exfiltration from the user's inbox/files via attacker-controlled URLs or markdown image links, unauthorized use of connected tools (e.g., Calendar.create from a malicious invite), unauthorized deletion of events, control of smart-home devices via Gemini Voice Assistant, and 'memory poisoning' of long-term Gemini memory. This is not a traditional software memory-safety bug; it is an inherent property of LLM agents that treat retrieved data as contextually trusted.
