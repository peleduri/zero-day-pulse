Ransomware Actors Exploit Unpatched SimpleHelp Remote Monitoring and Management to Compromise Utility Billing Software Provider

CVE-2024-57727 · CISA US-CERT Alerts · Thu, 12 Ju

Summary The Cybersecurity and Infrastructure Security Agency (CISA) is releasing this advisory in response to ransomware actors leveraging unpatched instances of a vulnerability in SimpleHelp Remote Monitoring and Management (RMM) to compromise customers of a utility billing software provider. This incident reflects a broader pattern of ransomware actors targeting organizations through unpatched v…

✨ Parallel AI Enrichment
Technical DetailsCVE-2024-57727 is an unauthenticated directory/path traversal (CWE-22) in the SimpleHelp RMM server. Crafted HTTP requests allow an unauthenticated remote attacker to read arbitrary files, including serverconfig.xml, which contains hashed administrator passwords and other secrets. The vulnerability is typically chained in a three-step attack: (1) CVE-2024-57727 path traversal is used to exfiltrate the server configuration and admin password hash; (2) the hash is cracked, or CVE-2024-57726 is used to abuse technician credentials, to obtain administrative access; and (3) CVE-2024-57728 is leveraged to achieve arbitrary code execution on the SimpleHelp server. Attackers then push unsolicited SimpleHelp technician client installations to downstream MSP customers for lateral movement and ransomware deployment.
Affected ProductsSimpleHelp Remote Monitoring and Management (RMM) versions 5.5.7 and all earlier releases (including 5.5.0-5.5.7, 5.4.x and earlier, 5.3.x and earlier). Related vulnerabilities CVE-2024-57726 (improper access control) and CVE-2024-57728 (server-side code execution) are commonly chained with CVE-2024-57727.

CVSS VectorCVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:N/A:N

Active ExploitationYes - confirmed active exploitation in the wild. CISA added this CVE to the Known Exploited Vulnerabilities (KEV) Catalog on February 13, 2025, and published Advisory AA25-163A on June 12, 2025 documenting ransomware actors exploiting unpatched SimpleHelp RMM instances since January 2025 to compromise a utility billing software provider and its downstream customers in a double-extortion ransomware attack (DragonForce). Independent reporting includes Arctic Wolf (January 22, 2025), The Hacker News (February 7, 2025), Help Net Security (May 28, 2025), and Sophos (May 27, 2025).
Threat ActorsDragonForce ransomware actors (per CISA Advisory AA25-163A). MitigationPer CISA Advisory AA25-163A and SimpleHelp: (1) Immediately upgrade SimpleHelp servers to version 5.5.8 or later (or 5.4.10 / 5.3.9 on older branches). (2) For unpatched instances, isolate the SimpleHelp server from the internet or stop the SimpleHelp server process. (3) Change the SimpleHelp Administrator password and all Technician account passwords (unless authenticated via SSO/AD), and rotate any API tokens. (4) Restrict at the firewall which IP addresses may reach the SimpleHelp server, which may log in as Technician/Administrator, and which may call APIs. (5) Create Server Event alerts for Administrator logins, failed login attempts, and configuration changes. (6) Audit Remote Access Services for unrecognized server URLs. (7) Threat-hunt for suspicious executables with three-alphabetic-letter filenames (e.g., aaa.exe, bbb.exe) created after January 2025. (8) Maintain offline/clean backups and do not expose services such as RDP directly to the internet.

✨ Parallel AI Enrichment
Technical DetailsIndirect prompt injection (IPI) is an attacker technique against LLM-integrated applications such as Workspace with Gemini. The attacker plants adversarial natural-language instructions inside any external data the model ingests while servicing a user query — emails, documents, calendar invites, web pages, screenshots, or tool/connector outputs. When the LLM reads that untrusted content as part of its context, the embedded instructions override or augment the system/user prompt, causing the model to follow the attacker's intent instead of (or in addition to) the legitimate user request. IPI does not require any input directly from the victim user and survives the application boundary. Demonstrated downstream effects include bypassing Gemini privacy/safety controls (Miggo calendar-invite attack, Jan 19 2026), generating convincing phishing content inside Gemini for Workspace (HiddenLayer, Sep 25 2024), exfiltrating data through tool calls (Unit 42 AI-agent telemetry), and executing tool actions on the victim's authenticated session in agentic AI workflows.

✨ Parallel AI Enrichment
Technical DetailsIndirect prompt injection (IDPI) is the primary new threat facing agentic browsers. An attacker embeds hidden or disguised instructions into web content that the Gemini in Chrome agent will ingest as part of its browsing task (e.g., as user-generated content such as product reviews, third-party iframes, or content the user asked the agent to read). Common delivery techniques include CSS-based hiding (display:none, visibility:hidden, opacity:0, white-on-white camouflage, font-size:0, off-screen positioning), HTML attribute cloaking, XML/SVG CDATA encapsulation, and Base64-encoded runtime assembly with timed delays. When the page content reaches the Gemini planning model, the hidden instructions are interpreted as legitimate commands, causing the agent to take attacker-controlled actions such as exfiltrating sensitive page data (form fields, tokens, account page contents), performing unauthorized transactions (e.g., banking actions), accessing local files or credentials, manipulating browser state, or accepting phishing instructions. Because the agent has both read access to the page DOM and the ability to take privileged actions on behalf of the user, IDPI bypasses traditional sandboxing that assumes user instructions and page content are trustworthy.
Affected ProductsGoogle Chrome with the Gemini in Chrome / agentic browsing integration. Gemini in Chrome began rolling out to all Mac and Windows desktop users in the United States on 2025-09-18, and the 'Auto Browse' agentic capability is available to Google AI Pro and AI Ultra subscribers in the United States. The blog post does not specify a numeric Chrome version cut-off; the architectural defenses described apply to the rolling-out / upcoming agentic capabilities in Chrome.

MitigationGoogle's announced defense-in-depth architecture for Gemini in Chrome agentic browsing (per the Dec 8, 2025 advisory and corroborating coverage) comprises: (1) User Alignment Critic — a separate, isolated Gemini model that vets proposed agent actions against the user's original intent using metadata only (no exposure to untrusted web content), acting as a 'high-trust system component'; (2) Agent Origin Sets (origin isolation) — categorizes web origins into 'read-only' (data may be consumed) and 'read-writable' (data may be sent/form-submitted to), with strict no-cross-isolation gating to bound cross-origin data leakage; (3) deterministic user confirmations — required for sensitive actions (banking/financial sites, medical sites, Google Password Manager sign-ins) and for first-encountered origins; (4) a real-time prompt-injection classifier running in parallel with the planning model's inference to block unaligned actions; (5) continuous automated and human red-teaming to discover new IDPI variants; (6) Spotlighting research (arXiv 2403.14720) combined with Chrome's existing Site Isolation and same-origin policy. User hardening advice (per Google's regular Chrome guidance) is to keep Chrome fully updated, enable Enhanced Safe Browsing / Google Password Manager, and avoid installing untrusted extensions that can interact with the Gemini panel.

When the user subsequently issues a benign query, the LLM conflates data and instructions and is steered to perform attacker-chosen actions such as exfiltrating the user's private context (summaries, calendar data, mail contents) through outbound tool calls (e.g., Calendar.create, Send email, render image with attacker-controlled URL) that the user has authorised. Unlike direct prompt injection, IPI requires no direct attacker access to the prompt and no user interaction beyond receiving the booby-trapped document. Google's post notes that Gemini 2.5 model training with adversarial data and the full layered stack significantly enhanced defenses against indirect prompt injection attacks in Gemini 2.5 models.
