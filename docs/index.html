Technical DetailsUnauthenticated path traversal (CWE-22) in the SimpleHelp web server. Multiple endpoints (e.g., the Toolbox / file-fetch endpoints) fail to properly sanitize user-supplied path sequences, allowing an unauthenticated remote attacker to submit crafted HTTP requests using directory-traversal sequences (e.g., '../') to download arbitrary files from the SimpleHelp host. ✨ Parallel AI Enrichment
Technical DetailsIndirect prompt injection (IPI) is a vulnerability class in LLM-based systems where an attacker embeds malicious natural-language instructions inside external content that an AI assistant later retrieves, ingests, or is otherwise exposed to (e.g., emails, documents, calendar invitations, web pages, shared files, notifications, or voice/text assistant messages). Because the LLM cannot reliably distinguish trusted system/user instructions from instructions parsed out of untrusted data, the hidden prompt is treated as authoritative and executed alongside the legitimate task. This can coerce the model to take unauthorized actions such as exfiltrating data, sending messages, modifying or deleting content, performing financial transactions, granting permissions, or pivoting to other connected tools. Common IPI vectors include hidden text, zero-width/white-on-white text, transparent images, HTML comments, page metadata, calendar fields, and indirect payloads delivered via messaging/notification surfaces. Successful exploitation often requires no direct user input beyond triggering the AI to process the poisoned content (a 'zero-click' style flow).

MitigationGoogle has deployed a layered, continuously updated defense strategy for Workspace with Gemini rather than a single patch. Mitigating prompt injection attacks with a layered defense strategy

No CVE · Google Security Blog · 2025-06-13

Posted by Adam Gavish, Google GenAI Security Team With the rapid adoption of generative AI, a new wave of threats is emerging across the industry with the aim of manipulating the AI systems themselves. One such emerging attack vector is indirect prompt injections. Unlike direct prompt injections, where an attacker directly inputs malicious commands into a prompt, indirect prompt injections involve…

✨ Parallel AI Enrichment
Technical DetailsIndirect prompt injection (IPI) is a class of attack against generative AI systems in which adversaries embed hidden malicious instructions inside external data sources that the AI subsequently retrieves or processes (e.g., emails, documents, calendar invites, webpages, files, or even images). Because modern LLMs/agents treat retrieved content and natural-language instructions interchangeably, the model interprets the attacker-supplied text as legitimate system/user-level instructions, causing it to deviate from the user's original intent. Typical attack outcomes include: (1) data exfiltration — the AI is manipulated into rendering an outbound link (e.g., an <img> tag pointing to an attacker-controlled server) or constructing a URL that carries sensitive context (conversation, emails, files) to an external host; (2) tool/agent abuse — the AI invokes connected tools (search, email send, file delete) on the attacker's behalf; (3) instruction override / jailbreak — the hidden prompt suppresses safety guardrails. Compared to direct prompt injection (where the attacker types malicious text directly into the prompt), indirect prompt injection is more dangerous because the malicious payload lives in third-party content the user cannot easily inspect, requires zero user clicks in many cases, and scales by poisoning any data source the AI consumes.
Affected ProductsGemini app (consumer Gemini), Gemini in Google Workspace (Gmail, Google Docs/Editors, Google Drive, Google Chat), Gemini 2.5 models

MitigationGoogle's layered defense strategy documented in the advisory comprises six reinforcing layers: (1) Gemini 2.5 model hardening via adversarial training with synthetic prompt-injection data; (2) Prompt-injection content classifiers — proprietary ML models that score incoming content for injection attempts; (3) Security thought reinforcement — targeted system-prompt instructions that remind the model to ignore instructions found in data; (4) Markdown sanitization and suspicious-URL redaction — rendered content is stripped of dangerous markdown and URLs are checked against Google Safe Browsing; (5) User confirmation framework (human-in-the-loop) — risky/sensitive actions triggered by model behavior require explicit user confirmation before execution; (6) End-user security mitigation notifications — users are informed when potentially malicious content was blocked. Technical DetailsCVE-2023-23397 is an Elevation of Privilege / NTLM credential-theft vulnerability in Microsoft Outlook for Windows. An attacker sends a specially crafted email containing a TNEF message that abuses the extended MAPI property PidLidReminderFileParameter to specify a custom reminder sound on an attacker-controlled UNC path (e.g., \\attacker.com\share\file.wav). When Outlook processes the message, it automatically connects to that remote SMB share, forcing the victim's machine to authenticate and leak the user's Net-NTLMv2 hash to the attacker's server. The attack requires no user interaction (no click or preview needed), and the captured hash can be relayed for authentication or cracked offline to recover the plaintext password, enabling lateral movement and account takeover.

Technical DetailsCWE-78 OS Command Injection. An unauthenticated remote attacker can send a specially crafted HTTP POST request to the Management API endpoint /mics/api/v2/sentry/mics-config/handleMessage (the ConfigServiceController 'handleMessage' endpoint). The endpoint parses a user-supplied XML 'message'/'commandexec' payload as an internal configuration command without requiring authentication and without adequate input validation, allowing arbitrary OS commands to be injected and executed by the appliance with root privileges. Successful exploitation grants the attacker full control over the Ivanti Sentry appliance, enabling configuration changes, credential theft, exposure of session tokens, impersonation of users to access downstream enterprise applications and email servers, and lateral movement into the internal network.
