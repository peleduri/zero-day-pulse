Summary The Cybersecurity and Infrastructure Security Agency (CISA) is releasing this advisory in response to ransomware actors leveraging unpatched instances of a vulnerability in SimpleHelp Remote Monitoring and Management (RMM) to compromise customers of a utility billing software provider. This incident reflects a broader pattern of ransomware actors targeting organizations through unpatched v…

✨ Parallel AI Enrichment
Technical DetailsAn unauthenticated path traversal (directory traversal) flaw exists in the SimpleHelp web application's '/toolbox-resource' endpoint, where the 'filepath' parameter is not properly sanitized. A remote, unauthenticated attacker can issue a crafted HTTP request containing traversal sequences (e.g., '../') to escape the web server's document root and download arbitrary files from the SimpleHelp host. Sensitive files retrievable include 'serverconfig.xml' (containing hashed passwords and credentials), '/etc/passwd', Linux SSH private keys (e.g., '/root/.ssh/id_rsa'), and Windows SAM database files. The leaked secrets enable downstream compromise of the RMM server and its managed customers.
Affected ProductsSimpleHelp Remote Monitoring and Management (RMM) / Remote Support software, versions 5.5.7 and earlier (all releases prior to 5.5.8).

Sources: https://www.rapid7.com/db/modules/auxiliary/scanner/http/simplehelp_toolbox_path_traversal/ ; https://www.horizon3.ai/attack-research/disclosures/critical-vulnerabilities-in-simplehelp-remote-support-software/ ; https://offsec.com/blog/cve-2024-57727

(2) Until upgraded, isolate SimpleHelp servers from the open internet or stop the SimpleHelp service. ✨ Parallel AI Enrichment
Technical DetailsIndirect Prompt Injection (IPI) occurs when an AI/LLM-based system processes external, attacker-controlled content such as a webpage, email, document, image, or file. That content contains hidden natural-language or encoded instructions that the model treats as legitimate prompts. When the model ingests the poisoned content, it follows the attacker's commands instead of, or in addition to, the original user/system intent. Attack vectors documented in the wild include: (1) hidden instructions on public web pages that cause AI agents to alter conversational tone, exfiltrate data via image-markdown URLs (EchoLeak), or stream infinite text to DoS the agent; (2) instructions that try to delete files on the user's machine or run shell commands via agentic coding tools (Cursor, GitHub Copilot, Claude Code); (3) SEO manipulation instructing AI assistants to promote the attacker's site; (4) financial fraud payloads that route AI-driven actions to attacker-controlled PayPal/Stripe endpoints; and (5) magic-string spoofing that impersonates target agent prompts. Google's Common Crawl analysis showed a 32% relative increase in the malicious-payload category between November 2025 and February 2026.
Affected ProductsLLM-powered AI agents and assistants generally; specifically named: Google Gemini, Google Workspace (Gemini in Workspace), GitHub Copilot, Cursor, Claude Code (Anthropic), Microsoft 365 Copilot (CVE-2025-32711 EchoLeak), AI browsers such as OpenAI Atlas/Perplexity Comet, and LLM-powered email assistants, customer-support chatbots, RAG applications, and multimodal AI systems. Specific product versions are not enumerated because Indirect Prompt Injection is a vulnerability class (OWASP LLM01:2025) affecting all LLM systems that ingest untrusted external content rather than a single defect with a fixed version range.

CVSS VectorCVSS vector unavailable. Indirect Prompt Injection is a vulnerability class (OWASP LLM01:2025) rather than a single defect, so no single CVSS v3 vector applies. Individual instances (e.g., CVE-2025-32711 in Microsoft 365 Copilot) may have their own vectors, but none is associated with the Google blog post referenced.

Threat ActorsNone known. For Microsoft 365 Copilot, a server-side fix for EchoLeak (CVE-2025-32711) was deployed by Microsoft in May 2025.

✨ Parallel AI Enrichment
Technical DetailsIndirect prompt injection (IPI) is a vulnerability of generative-AI systems in which an attacker plants hidden malicious instructions inside untrusted external content (emails, documents, calendar invites, web pages, files, images) that the LLM later ingests as part of its context. Because the LLM cannot reliably distinguish data from instructions, it can be coerced into ignoring the legitimate user task and instead executing attacker-supplied commands — exfiltrating user data (e.g. summarizing and forwarding private email or calendar contents to an external destination), performing unauthorized actions (deleting calendar events, sending messages, executing payments), or disclosing hidden system prompts. The vector is 'indirect' because the malicious prompt is never typed by the user; it travels into the model via external content the user merely asks the assistant to read or summarize.
