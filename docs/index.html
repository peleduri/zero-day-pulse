Technical DetailsIndirect Prompt Injection (IPI) is a prompt-injection variant where an attacker embeds adversarial instructions inside content that an LLM/AI agent ingests as data — including web pages, emails, documents, HTML comments, CSS-hidden elements (display:none, font-size:1px, rgba(0,0,0,0)), aria-hidden attributes, meta tags, or README/code comments. ✨ Parallel AI Enrichment
Technical DetailsIndirect Prompt Injection (IPI) is a class of attack against LLM-based applications such as Workspace with Gemini. An attacker plants hidden instructions inside content that the LLM will later retrieve or process — for example, a malicious instruction hidden in a Gmail message, a shared Google Docs/Drive file, a Calendar invite, or a third-party webpage. When Gemini ingests that content (often via Retrieval-Augmented Generation), the embedded instructions are treated as if they came from the user, allowing the attacker to override the user's intent, coerce the model to invoke tools (search, send mail, run code, call APIs), exfiltrate sensitive data, or take destructive actions — sometimes with no direct user input (zero-click). The GeminiJack variant exploited the trust boundary in a RAG architecture between user-controlled content in Workspace data sources and the Gemini Enterprise instruction channel.

Technical DetailsIndirect prompt injection (IDPI) embeds hidden or malicious instructions inside web content that an AI agent later ingests. Adversaries use CSS cloaking (display:none, off-screen positioning, white-on-white text), HTML attribute cloaking (data-* attributes), XML/SVG CDATA encapsulation, or base64 payloads decoded at runtime with timed delays. When the agentic browser's Gemini model processes the poisoned page (e.g., to summarize, fill a form, or perform an agentic task on the user's behalf), the embedded instructions are treated as legitimate commands, enabling unauthorized financial transactions, exfiltration of sensitive data, or navigation to attacker-controlled URLs. The GeminiJack variant is an indirect prompt injection inside the Gemini Enterprise RAG pipeline where attackers poison Google Docs, Calendar invites, or Gmail messages so that Vertex AI Search retrieves them as instructions and exfiltrates sensitive corporate data via an outbound HTTP request.

Posted by Adam Gavish, Google GenAI Security Team With the rapid adoption of generative AI, a new wave of threats is emerging across the industry with the aim of manipulating the AI systems themselves. One such emerging attack vector is indirect prompt injections. Unlike direct prompt injections, where an attacker directly inputs malicious commands into a prompt, indirect prompt injections involve…

✨ Parallel AI Enrichment
Technical DetailsIndirect prompt injection embeds hidden malicious instructions inside external content (emails, documents, calendar invites, web pages) that an LLM-powered AI assistant later ingests during normal operation. Unlike direct prompt injection (where the user types the malicious prompt), the victim never composes the attack — the malicious instruction arrives via ordinary data-channel content. Because the model cannot reliably distinguish trusted system instructions from untrusted data-channel content, the hidden instructions can cause the assistant to exfiltrate user data (e.g., email contents), perform unauthorized tool calls, or take other rogue actions. Google's blog describes a layered defense: (1) Gemini 2.5 model hardening via adversarial training data, (2) proprietary ML-based prompt-injection content classifiers, (3) system-instruction reinforcement appending security directives, (4) markdown sanitizer stripping external image URLs, (5) Google Safe Browsing-based suspicious URL detection/redaction, (6) Human-In-The-Loop (HITL) confirmation prompts for risky actions, and (7) contextual end-user security notifications. Google's companion posts add red-teaming via the Simula synthetic-data framework, the Google AI Vulnerability Rewards Program, deterministic defenses via a centralized policy engine, and DeepMind techniques such as 'spotlighting,' 'self-reflection,' and ART-based fine-tuning.
Affected ProductsGemini 2.5 LLM family, Google Gemini app, and Gemini integrations in Google Workspace (Gmail, Google Calendar, Google Docs editors, Google Drive, Google Chat). No specific patch version numbers are published.
