Summary The Cybersecurity and Infrastructure Security Agency (CISA) is releasing this advisory in response to ransomware actors leveraging unpatched instances of a vulnerability in SimpleHelp Remote Monitoring and Management (RMM) to compromise customers of a utility billing software provider. This incident reflects a broader pattern of ransomware actors targeting organizations through unpatched v…

✨ Parallel AI Enrichment
Technical DetailsCWE-22: Improper Limitation of a Pathname to a Restricted Directory ('Path Traversal'). SimpleHelp Remote Support/RMM versions 5.5.7 and earlier contain multiple path-traversal flaws that allow unauthenticated remote attackers to download arbitrary files from the SimpleHelp host via specially crafted HTTP requests. Readable files include the server configuration file containing secrets and hashed user (admin) passwords. Attackers retrieve the admin password hash, crack it offline, then chain it with other flaws (e.g., CVE-2024-57726 upload) to obtain remote code execution as a technician/SYSTEM, which they use to deploy ransomware to the RMM host and pivot to downstream MSP customers.
Affected ProductsSimpleHelp Remote Monitoring and Management (RMM) / Remote Support software, version 5.5.7 and all earlier releases (including 5.4.x and 5.3.x branches).

Patch AvailableYes - SimpleHelp released patches in January 2025, approximately two days after responsible disclosure and one week before public disclosure. ✨ Parallel AI Enrichment
Technical DetailsIndirect Prompt Injection (IPI) attacks LLM-powered agents and assistants by causing untrusted external content (web pages, emails, documents, calendar invites, search-query history) to be processed as if it were direct instructions, overriding the system prompt's intended behavior. Adversaries defeat the data-vs-instruction boundary using techniques such as: invisible / off-screen text (1-pixel fonts, near-transparent colors, zero-width characters, aria-hidden / visually-hidden DOM nodes), instructions buried in HTML comments and page metadata, Base64-encoded instructions decoded at runtime, canvas-rendered text invisible to text analyzers, timed delays to evade scan pipelines, multilingual or JSON/code-style obfuscation, semantic social-engineering phrases ('Ignore previous instructions'), and JavaScript-triggered multi-search-query injection poisoning victim search history for subsequent personalization ingestion. Documented in-the-wild payloads include: tone-change pranks, SEO promotion, 'deterrence' payloads luring AI crawlers into infinite-stream pages, system-prompt leakage, sensitive-data exfiltration (memory, location, credentials, API keys), forced user redirects/phishing, instruction hijack, and destructive actions.
Affected ProductsAI agents and LLM-powered assistants broadly; specifically Google Gemini (including Gemini Search Personalization), Google Bard, Microsoft Bing AI/Copilot, OpenAI ChatGPT, AI-powered indexers/crawlers, AI ad-review systems. No specific affected versions were published.

(4) Forcepoint X-Labs published 10 in-the-wild payload samples (https://forcepoint.com/blog/x-labs/indirect-prompt-injection-payloads).

Threat ActorsNone known

(2) Hierarchy of instructions, adversarial training, and pressure-testing red-teaming (Google runs continuous AI red-teaming and an AI Vulnerability Reward Program). (3) Real-time, web-scale detection (intent analysis, prompt-visibility assessment, behavioral correlation) using telemetry signals like Common Crawl. (4) Content Security Policy hardening, input validation/sanitization, and limiting agent capabilities. (5) Human-in-the-loop for destructive or financially significant actions. (6) Network-level / proxy-level blocking of IPI URLs (Forcepoint Real-Time Analytics; Palo Alto Networks Advanced URL Filtering, Advanced DNS Security, Prisma AIRS, Prisma Browser). (7) For users of Gemini-style personalization, clearing or disabling personalization history reduces the search-history-poisoning attack surface demonstrated in Tenable TRA-2025-23.

✨ Parallel AI Enrichment
Technical DetailsIndirect prompt injection (IPI) is a class of attack against LLM-based applications where an adversary plants malicious instructions inside external data sources (emails, calendar invites, documents, web pages, notifications, images) that the LLM ingests while fulfilling a user's request. The model cannot reliably distinguish trusted system prompts from untrusted data. Obfuscation methods include CSS hiding (display:none, opacity:0, font-size:0, off-screen positioning), HTML/XML/SVG attribute cloaking, metadata injection, system-prompt tag impersonation, and conditional targeting ('If you are an AI/LLM...'). Representative attacks: SafeBreach's notification-based hijack of Gemini Voice Assistant; Miggo's Calendar invite payload causing data exfiltration to attacker-controlled events; HiddenLayer's payload in Gmail/Docs/Slides/Drive for phishing content manipulation; and Unit 42's web-based IDPI campaigns using hidden HTML/metadata for SEO manipulation, ad-review evasion, data exfiltration, and system-prompt leakage.
Affected ProductsGoogle Workspace with Gemini (Gmail, Google Docs/Editors, Google Drive, Google Calendar, Google Slides, Google Chat), standalone Gemini app, Gemini Voice Assistant (Android Utilities agent / notification surfaces), Gemini model versions 1.5, 2.0, and 2.5
CVSS Score10.0
CVSS VectorCVSS vector unavailable.
Exploit AvailableYes — multiple public PoCs and weaponized demonstrations exist. Sources: SafeBreach Labs (https://www.safebreach.com/blog/gemini-voice-assistant-prompt-injection-exploit/), Miggo Research (https://www.miggo.io/post/weaponizing-calendar-invites-a-semantic-attack-on-google-gemini), HiddenLayer (https://www.hiddenlayer.com/research/new-gemini-for-workspace-vulnerability-enabling-phishing-content-manipulation), 0Din (https://0din.ai/blog/phishing-for-gemini), and Atlas PoC (https://github.com/brennanbrown/atlas-prompt-injection-poc).
Patch AvailableNo single discrete patch URL — Google treats IPI mitigation as a continuous, layered program rather than a one-time fix. Incremental 'point fixes' include regex takedowns, configuration updates, Gemini model retraining, content-classifier updates, and Delayed Tool Invocation controls. Responsible-disclosure collaborations with SafeBreach Labs and Miggo Research led to fixes for Calendar and notification vectors. HiddenLayer-reported Gmail/Docs/Slides/Drive issues were marked 'Won't Fix (Intended Behavior)'. References: https://security.googleblog.com/2026/04/google-workspaces-continuous-approach.html and https://blog.google/security/prompt-injections-web/
Active ExploitationYes — confirmed active exploitation in the wild has been reported. (1) Unit 42 / Palo Alto Networks 'Web-Based Indirect Prompt Injection Observed in the Wild' (Mar 3, 2026) documented a December 2025 real-world instance where hidden IDPI instructions bypassed an AI ad-review system. (2) Forcepoint X-Labs '10 Indirect Prompt Injection Payloads Caught in the Wild' (Apr 22, 2026) catalogued 10 live IPI payloads including API-key exfiltration, SEO hijacking, data destruction, and financial fraud. (3) Google's own monitoring sweep (Apr 23, 2026) observed payloads including harmless pranks, SEO manipulation, AI-agent deterrence, data exfiltration, and data destruction. OWASP ranks prompt injection as the #1 LLM security risk in 2026.
Threat ActorsNone known (no specific APT group, ransomware operator, or named threat-actor campaign has been publicly attributed to this vulnerability class). Publicly documented exploitation comes from independent security researchers (SafeBreach Labs, Miggo, HiddenLayer, 0Din, Unit 42, Forcepoint X-Labs) and unattributed in-the-wild actors.
MitigationGoogle's layered defense includes: (1) Deterministic defenses — user-confirmation prompts before sensitive tool calls, URL sanitization, strict tool-chaining policies, centralized Policy Engine for regex takedowns; (2) ML-based defenses — retraining Gemini with synthetic IPI data and adversarial fine-tuning; (3) LLM-based defenses — refined system instructions, spotlighting/paraphrasing, and 'LLM instruction pointer' to separate instructions from data; (4) Model hardening; (5) Proactive discovery via red-teaming (TAP, Beam Search, Actor-Critic) and AI Vulnerability Reward Program; (6) Coarse-to-fine detection pipeline (pattern matching → LLM classification → human validation); (7) Content-classifier updates and 'Delayed Tool Invocation' controls. Hardening guidance: avoid unchecked tool/function-call authority over sensitive data, treat all retrieved content as untrusted, sanitize HTML before LLM ingestion, require explicit user consent for outbound actions.

Technical DetailsCVE-2025-48530 is a linear buffer overflow in CrabbyAVIF, the AVIF image decoder that Google adopted in Android using Rust. Technical DetailsIndirect prompt injection embeds malicious natural-language instructions in untrusted external content (emails, documents, calendar invites, web pages, PDFs, images) that the LLM later retrieves and processes.
