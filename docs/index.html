Technical DetailsIndirect Prompt Injection (IPI) occurs when adversarial instructions are embedded in external content (web pages, documents, search results, or retrieved context) that an LLM or agent ingests via retrieval or browsing. Attackers craft payloads that blend into benign content to bypass simple filters and rely on the model's instruction‑following behavior to execute the malicious directive. IPI can target RAG systems and agentic workflows that retrieve and incorporate external content into prompts; techniques demonstrated include poisoning web content, crafting natural‑language injection patterns, and exploiting retrieval pipelines and tool‑invocation logic.
