Technical DetailsSeven flaws in ChaN FatFs R0.16 and earlier, all triggered by a crafted FAT/exFAT/GPT image delivered via removable media (USB/SD) or OTA channels. (1) CVE-2026-6682 — integer overflow in FAT32 mount_volume (fasize *= fs->n_fats wraps), CWE-190; memory corruption/code execution. (2) CVE-2026-6683 — divide-by-zero in exFAT sync/write when n_fatent-2 == 0, CWE-369; device crash / OTA bricking. (3) CVE-2026-6684 — unbounded GPT partition-scan loop (GPTH_PtNum trusted without cap), CWE-835; DoS/hang. (4) CVE-2026-6685 — unsigned-subtraction wrap in dirty-cache handling (fp->sect - sect < cc), CWE-191; stale-cache OOB, silent data corruption. (5) CVE-2026-6686 — uninitialized clusters on f_lseek beyond EOF, CWE-908; data leakage. (6) CVE-2026-6687 — stack-based buffer overflow in f_getlabel (XDIR_NumLabel unbounded), CWE-121; arbitrary code execution. (7) CVE-2026-6688 — long-filename overflow in downstream wrappers copying fno.fname (up to 255 chars) into fixed-size buffers, CWE-120; buffer overflow in caller code. All attack vectors are Physical (AV:P).
Affected ProductsChaN FatFs R0.16 and earlier (R0.13 through R0.16b), shipped in firmware using: Espressif ESP-IDF, STMicroelectronics STM32Cube middleware, Zephyr RTOS, MicroPython, ArduPilot, RT-Thread, Arm Mbed, Samsung TizenRT, SWUpdate, Rene Stange's Circle, Adafruit TinyUF2, and Eugene Tarassov's Vivado RISC-V build. Posted by Adam Gavish, Google GenAI Security Team Indirect prompt injection (IPI) is an evolving threat vector targeting users of complex AI applications with multiple data sources, such as Workspace with Gemini. This technique enables the attacker to influence the behavior of an LLM by injecting malicious instructions into the data or tools used by the LLM as it completes the user’s query. This m…

✨ Parallel AI Enrichment
Technical DetailsIndirect Prompt Injection (IPI) is a class of attack in which an adversary embeds malicious natural-language (or hidden) instructions inside external content that an LLM later consumes—webpages scraped by an AI browser/summarizer, documents ingested by a RAG pipeline, or emails fetched through tool-use/function-calling. When the model retrieves or summarizes that content, the injected instructions override or subvert the user's original prompt, causing the agent to call tools or generate output the user did not request. Specific techniques include: payload splitting across documents (e.g., a resume), multimodal injection hidden in images, obfuscation (Base64, emoji, multi-language), adversarial suffixes, hidden text (white-on-white, visually-hidden CSS classes), code injection in function-calling outputs, and cross-document exfiltration where one document instructs Gemini to forward data from another. End-to-end chain Google documented: malicious trigger (e.g., 'Linear Generation' procedural text) embedded in email/document → Gemini retrieves data via tool use → trigger forces a function call that exfiltrates private user data (passwords, SSNs, passport numbers) to attacker-controlled endpoint. No direct user input required beyond the normal query that pulls untrusted content.
Affected ProductsGoogle Workspace with Gemini (Gemini integrated in Gmail, Google Docs editors, Google Drive, and Google Chat); standalone Gemini app; Gemini model family (Gemini 1.5, Gemini 2.0, Gemini 2.5). Broader class: any agentic LLM application that ingests untrusted external content (web pages, files, emails) including RAG pipelines, AI coding assistants, and AI browsing/summarization agents.

CVSS VectorCVSS vector unavailable.
Exploit AvailableYes — multiple public PoCs and in-the-wild payloads. SecurityWeek reported (April 2026) that Google's own broad sweep of the public web found increasing IPI activity. Independent researcher reports (HiddenLayer 2024, 0din.ai Jul 2025) demonstrated successful exploitation prior to these in-the-wild observations.
Threat ActorsNone known. No specific APT group, cybercriminal gang, or ransomware operator has been publicly attributed to exploiting indirect prompt injection (IPI) against Workspace with Gemini. Public reporting refers generically to attackers embedding instructions in untrusted content (web pages, emails, documents) and describes payload intents (API-key exfiltration, financial fraud, content suppression) but does not name specific threat actors.
MitigationGoogle employs a continuous, layered defense strategy: (1) Deterministic defenses—centralized Policy Engine enforcing user confirmation, URL/markdown sanitization, suspicious-URL redaction, and tool-chaining policies; (2) ML-based defenses—classifier models trained on synthetic IPI data expanded via the 'Simula' framework; (3) LLM-based defenses—refined system instructions (security thought reinforcement) pushing the model to ignore embedded directives; (4) Model resilience—retraining Gemini to internally identify and ignore harmful instructions in data, adversarial fine-tuning, in-context learning with positive/negative IPI examples, Retrieved-Data Classifier and User-Instruction Classifier. Recommended hardening for users/admins: ensure user confirmation prompts for risky operations are enabled, review end-user security-mitigation notifications surfaced by Gemini, restrict Gemini extensions/tools to a least-privilege set, separate untrusted external content via content-class boundaries, require human-in-the-loop approval for high-risk actions. OWASP LLM01:2025 guidance: least-privilege tool design (handle sensitive functions in code rather than via the model), output-format validation, semantic filtering, the RAG Triad (context relevance, groundedness, Q&A relevance), and regular adversarial testing.
