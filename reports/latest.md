**Parallel AI Enrichment:**

- **Technical Details:** Indirect Prompt Injection (IPI) exploits the trust boundary between an LLM's system/user prompt and the external data the LLM retrieves while answering a query. An attacker plants malicious natural-language (or encoded) instructions inside content the LLM will later read — emails, shared documents, calendar invites, web pages, HTML metadata, document footers, etc. When the user issues a benign query, the LLM pulls the poisoned content into its context window (often via Retrieval-Augmented Generation), cannot reliably distinguish user commands from data-borne instructions, and obeys the injected commands. Attacks can be zero-click (no user interaction required beyond a normal search/summarise, as in GeminiJack) and can target agentic capabilities (tool/function calling, code execution, email send, file deletion, payments). Obfuscation techniques observed in the wild include CSS hiding (display:none, visibility:hidden, opacity:0, zero-sized text, off-screen positioning), white-on-white text, HTML comments, XML/SVG CDATA sections, data-* attribute cloaking, base64-encoded payloads decoded at runtime, and meta-tag namespace injection. Data exfiltration is typically achieved by coercing the model to issue HTTP requests (e.g., image tags) to attacker-controlled servers.

**Parallel AI Enrichment:**

- **Technical Details:** Indirect prompt injection is a class of attack in which an adversary embeds hidden malicious instructions inside external content that an AI assistant ingests on the user's behalf (e.g., emails, documents, calendar invites, web pages, images). When the LLM processes that content together with the user's prompt, it cannot reliably distinguish attacker-controlled instructions from legitimate user intent, allowing the attacker to coerce the model into exfiltrating sensitive data (location, memory, contacts, calendar details), invoking tools/connectors without authorization, modifying or deleting user data, or performing other rogue actions. Unlike direct prompt injection (where the attacker types the malicious prompt themselves), the attack is delivered through benign-looking data sources the user has asked the AI to summarize or act on. Real-world examples include: (1) a calendar invite whose description contains a natural-language payload instructing Gemini to summarize meetings and call Calendar.create, leaking private schedule data (Miggo); (2) a malicious web page that uses JavaScript top-level navigation to seed fake queries into a user's Gemini search-history personalization model, then exfiltrates location/name/memory via markdown link rendering (Tenable TRA-2025-23).
