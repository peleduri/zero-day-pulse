**Parallel AI Enrichment:**

- **Technical Details:** Indirect prompt injection (IPI) is a class of attack against LLM-backed applications (e.g., Workspace with Gemini) where an adversary embeds malicious natural-language instructions inside a data source or tool that the LLM consumes while completing the user's request (e.g., an email body, Google Doc/Slide, Drive file, web page, or a Cloud log line). The LLM treats those untrusted instructions as legitimate directives, causing it to ignore the user's intent and instead (a) generate attacker-controlled output (phishing messages, malicious URLs, fake security alerts), (b) take unauthorized actions through connected tools (sending mail, modifying Drive/Docs assets, invoking Cloud integrations), or (c) exfiltrate conversation or workspace data. Concretely demonstrated techniques include hidden HTML/CSS (zero-sized text, display:none, opacity:0, color camouflage), SVG/CDATA cloaking, JavaScript-rendered prompts, and URL fragment (#) injection (e.g., 'HashJack'). Per Google's blog, IPI can succeed even without any direct user input, and it is amplified by agentic automation that combines many tools and data sources.
- **Affected Products:** Google Workspace with Gemini (Gemini app, plus Gemini integrations in Gmail, Google Docs, Google Slides, Google Drive, and Google Chat); Gemini 2.5 models; Google Cloud Gemini Cloud Assist. (Indirect prompt injection is a class of attack against AI applications with multiple data sources and agentic tooling; beyond Workspace, it broadly impacts LLM-backed agents that ingest untrusted data from web pages, logs, and third-party tools.)
- **CVSS Score:** 7.5
- **CVSS Vector:** CVSS vector unavailable.
- **Exploit Available:** Yes. **Parallel AI Enrichment:**

- **Technical Details:** Indirect prompt injection (IPI) attacks exploit the architecture of generative AI assistants like Gemini, which ingest external/untrusted content (emails, documents, calendar invites, web pages) as part of fulfilling user requests.
